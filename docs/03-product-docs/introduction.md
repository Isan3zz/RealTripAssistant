# RealTrip Assistant — 产品介绍

> 你的私人旅行规划助手，不做预订，只做方案。

---

## 一句话定位

RealTrip Assistant 是一个**非交易型智能旅行规划助手**。它帮你完成旅行中最消耗精力的部分——信息收集、行程编排、方案比较——然后把最终方案交给你去执行。

---

## 为什么需要它？

做一次旅行规划通常要反复做这些事：

- 在多个 App 之间切换查景点、天气、交通
- 手动排时间线，总是算不清活动间距够不够
- 不知道预算花在哪，最后发现超支
- 想调整一处，结果整天的安排都要重排
- 旅途中发现计划不合理但没有备选方案

RealTrip Assistant 把这些事自动化，让你从"规划执行者"变成"方案决策者"。

---

## 核心能力

### 智能行程生成

你只需要说"杭州 4 天，带老人，慢节奏，预算 2 万"，系统自动生成完整的每日行程——包含活动安排、时间线、费用估算。

### 多维校验保障

系统不是"生成了就完事"。每次生成的行程自动通过 8 层确定性规则校验（时间不重叠、预算不超限、交通缓冲充足……），再经过语义合理性判断（节奏是否匹配、动线是否合理），确保方案真实可用。

### 用户锁定

你对某个安排特别满意？一键锁定。后续任何修改都不会动你的锁定项。

### 方案对比

拿不定两个方案哪个好？系统给出并排对比，差异高亮、维对对，帮你快速决策。

### 偏好学习

系统记住你的偏好——但不是全局平均。"带老人出游时喜欢慢节奏"和"自己出门时喜欢深度游"会被区分记录，在不同上下文中自动适配。

### 全格式导出

方案确定后，一键导出为 Markdown、PDF、日历 ICS——自由编辑或分享。

---

## 为什么不做预订？

预订意味着：实时库存、支付、退改签——这些需要接入多个平台的后台交易系统，极大增加复杂度和维护成本。保持"规划层"定位让我们能专注于做好一件事：**生成高质量的旅行方案**。

系统生成的每个推荐都附有来源链接，你可以一键跳转到相应平台自行预订。

## 适用人群

| 人群 | 场景 |
|------|------|
| **自由行旅客** | 自己规划行程，但不想一个个查景点排时间 |
| **带老人/小孩的家庭** | 需要特殊节奏和强度控制的行程 |
| **多目的地旅行者** | 需要跨城市行程编排，算交通衔接 |
| **旅行规划爱好者** | 喜欢比较多个方案，选最优解 |

## 产品演进路线

| 阶段 | 状态 | 核心能力 |
|------|------|---------|
| **Phase 1 (MVP)** | ✅ 当前 | CLI 规划生成 + 4 条基础规则校验 |
| **Phase 2** | 📅 下一个 | 4 Agent 协同 + 完全校验体系 + 用户锁定 + 方案差异对比 |
| **Phase 3** | 📅 规划中 | Web UI + 多会话 + 偏好学习 + 导出中心 |
| **Phase 4** | 📅 远期 | 企业级：知识图谱 + 多模型路由 + 可观测体系 |

---

## 技术特色

- **规则优先**：能用代码校验的不用 LLM——规则引擎零 token 消耗、零延迟、100% 确定
- **证据驱动**：每个推荐都有来源，URL 可达性自动检查，杜绝幻觉链接
- **分级降级**：外部 API 失败时自动按策略降级，不因单个服务中断导致整个规划失败
- **结构化演进**：从 1 Agent 到 6 Agent，从文件系统到分布存储，每阶段可独立交付
